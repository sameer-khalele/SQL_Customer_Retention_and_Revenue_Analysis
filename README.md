# SQL_Customer_Retention_and_Revenue_Analysis

## Project Overview
This project analyzes subscription performance using customer, subscription, transaction, and support-ticket data. The objective was to measure retention, revenue quality, and operational signals that may influence churn.

## Key KPIs
- Customers: 1,200
- Active Subscriptions: 914
- Churn Rate: 23.8%
- Revenue: $399,091
- Average Revenue per Customer: $333
- Average CSAT: 2.99/5

## Main Findings
- The portfolio has 914 active subscriptions and a churn rate of 23.8%, which provides a clean base for cohort and retention tracking.
- Premium generated the highest total revenue, while country-level revenue was strongest in Saudi Arabia.
- Support issues related to feature request produced the lowest customer satisfaction score, suggesting a direct service-quality retention risk.

## Recommendations
- Track monthly churn by acquisition channel and plan tier to identify where retention programs will have the largest impact.
- Combine support-ticket resolution speed with churn outcomes to test whether service friction predicts subscription loss.
- Use cohort analysis to compare long-term value across acquisition channels instead of relying only on short-term revenue.

## Tools
SQL, SQLite, Excel / Power BI, retention analysis
